Заходим на на github аккаунт, создаем репозиторий, далее копируем ссылку на путь к нему.
Выбираем место, где хотим хранить наш репозиторий локально.
Далее в терминале
git clone + ссылка  - клонируем репозиторий
main - ветка на которой мы находимся
git status - состояние git, символизирует текстом, выделяет состояние файла красным или зеленым цветом, что символизирует о состоянии файлов.
git add - копирование файлов
git commit -m - комит с комментарием
git push - "запушим" синхронизируем наши изменения, они отобразятся на странице гит-хаб
ssh keygen -t rsa C - создание ключа
git fetch - просмотр внешних изменений 
git pull - "перетягивание" изменений на свой локальный репозиторий

Practice:
TXT
 1. Создать внешний репозиторий c названием TXT.
 2. Клонировать репозиторий TXT на локальный компьютер.
git clone https://github.com/YanSurma/TXT.git
 3. Внутри локального TXT создать файл “new.txt”.
touch new.txt
 4. Добавить файл под гит.
git status - красный цвет
git add new.txt
git status - зеленый цвет, отлично, файл готов отправке.
 5. Закоммитить файл.
git commit -m "add new.txt"
 6. Отправить файл на внешний GitHub репозиторий.
git push
 7. Отредактировать содержание файла “new.txt” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате TXT.
git status
git add new.txt
 8. Отправить изменения на внешний репозиторий.
git commit -m "modified new.txt"
git push
 9. Создать файл preferences.txt
touch preferences.txt
 10. В файл preferences.txt” добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате TXT.
vim preferences.txt
 11. Создать файл sklls.txt добавить информацию о скиллах которые будут изучены на курсе в формате TXT
touch skills.txt
vim skills.txt
 12. Сделать коммит в одну строку.
git status
git add skills.txt
git add preferences.txt
 13. Отправить сразу 2 файла на внешний репозиторий.
git commit -m "add preferences and skills"
git push
 14. На веб интерфейсе создать файл bug_report.txt.
 15. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
 16. На веб интерфейсе модифицировать файл bug_report.txt, добавить баг репорт в формате TXT.
 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
git fetch - видим изменения
git pull - добавим к себе
gir status
git commit -m
